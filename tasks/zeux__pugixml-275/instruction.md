XML parsing currently rejects or mishandles attributes whose values are enclosed in single quotes (e.g. `<node attr='1'/>`). This is problematic for XML generated by tools that prefer single quotes, and causes otherwise-valid documents to fail to parse correctly.

Update the XML parser so that attribute values can be enclosed by either double quotes (`"`) or single quotes (`'`). When parsing an element like:

```xml
<node attr='1' other='2'><child>text</child></node>
```

the resulting DOM should contain attributes `attr` and `other` with values `1` and `2` respectively, same as if the input used double quotes.

After successfully parsing single-quoted attributes, serialization via the normal document printing/saving APIs should produce standard XML output that uses double quotes for attribute values (i.e., printing the parsed document should yield `<node attr="1" ...>` rather than preserving single quotes). This should work consistently across common formatting modes (pretty-printed with newlines/indentation, raw formatting, and indentation modes that place attributes on new lines).
