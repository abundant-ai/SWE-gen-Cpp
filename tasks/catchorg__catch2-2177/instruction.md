Catch2 headers are intended to be self-sufficient: including any single public header in an otherwise-empty translation unit should compile successfully. Currently, some headers do not stand on their own (they rely on transitive includes or missing forward declarations), so compiling a translation unit that only does `#include <that_header.hpp>` fails. This makes it easy for missing includes/declarations to slip in unnoticed.

Add a development/testing build mode that generates and compiles “surrogate translation units” for Catch2 headers. A surrogate translation unit is a `.cpp` file whose entire content is a single `#include` of exactly one Catch2 header (excluding aggregator/helper headers such as `*_all.hpp`). When this mode is enabled, the build should automatically create surrogate `.cpp` files for all relevant headers and compile them as part of the build (e.g., via an executable target that links against the normal Catch2 test/main target). The feature must be controlled by a CMake option named `CATCH_BUILD_SURROGATES`, and it should be intended to run under development builds so normal user builds are not slowed down.

After enabling surrogate builds, ensure that every supported header compiles when included alone. If any header fails to compile standalone, update the headers so they are self-sufficient (e.g., by adding the missing includes or required declarations) until all surrogate translation units build cleanly.

Additionally, add an automated check in CI that validates surrogate translation units exist and are built for all applicable headers, so that newly added headers cannot regress by missing their surrogate coverage or by becoming non-self-sufficient.
