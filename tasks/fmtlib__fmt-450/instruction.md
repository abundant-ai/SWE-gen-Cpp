A new container-backed output facility is needed to support formatting directly into an external, standard-compliant container (e.g., `std::basic_string`, `std::vector<char>`) without first formatting into an internal buffer and then copying.

Implement a `fmt::internal::ContainerBuffer<Container>` that wraps a caller-provided container by reference and exposes a buffer-like interface. When constructed with an empty container, `size()` and `capacity()` must both report 0. Calling `reserve(n)` must ensure `capacity()` becomes `n` while leaving `size()` unchanged. Calling `resize(n)` must set both `size()` and `capacity()` to `n`.

The buffer must also support appending a character range into the underlying container via `append(const CharT* begin, const CharT* end)`, where `CharT` is `Container::value_type`. After appending, the underlying container’s contents must include the appended characters at the end, and the buffer’s `size()` must match the container’s size. For example, if the container initially holds `"Why so"` and you append the range corresponding to `" serious"`, the container must become `"Why so serious"`.

On top of this, implement `fmt::BasicContainerWriter<Container>`, a streaming-style writer that writes formatted output directly into the referenced container, supporting both narrow and wide character strings depending on `Container::value_type`. It must support chained `operator<<` with string literals and integers, such that writing `"The answer is "`, then `42`, then `"\n"` produces exactly `"The answer is 42\n"` for `std::string` and `L"The answer is 42\n"` for `std::wstring`.

`BasicContainerWriter` must also correctly handle the case where the container is non-empty at construction time (append mode). If constructed with a container containing `"The"` (size 3), then `out.size()` must initially be 3, and subsequent streaming must append without overwriting existing content, resulting in `"The answer is 42\n"` and `out.size()` reporting 17 afterward. The same append behavior must work for `std::vector<char>` containing the three characters `T`, `h`, `e`, where after writing the remaining text the vector’s size must become 17 and match `out.size()`.

The container types supported by these utilities must only rely on common container operations: `value_type`, `size()`, `resize()`, `operator[]` for element access, and contiguous storage. The API should work for both `std::basic_string<CharT>` and `std::vector<CharT>` for appropriate `CharT`.
